Quarta Aula

Vivenciando HTML

Hoje aprendemos sobre as tags <header/>, <img/> e <nav/>.

Header:

É responsável por agrupar todos os elementos que façam parte do cabeçalho do contexto onde está inserido, ou seja, se o header é um elemento diretamente filho da tag <body/> entendemos que esse é o cabeçalho da página, como foi o exemplo da aula. Mas há contextos em que podemos colocar a tag <header/> para ser o cabeçalho do conteúdo. Como no exemplo abaixo:

<main> <header> <h1>Título do conteúdo</h1> <h2>Subtítulo em formato de descrição resumida<h2/> </header> </main>

Outro exemplo possível seria ter o breadcrumb dentro dessa estrtura no lugar da tag <h2/>.

Nav:

É responsável por englobar a lista de links que formam o menu do site. Basicamente sua estrutura se repete e em muitos casos é como no exemplo abaixo:

<nav> <ul> <li> <a href="link"> texto do link </a> </li> </ul> </nav>

Img:

É responsável por renderizar imagens no html através do endereço do arquivo no projeto que é passado pelo atributo src, que é o único atributo obrigatório para que a imagem seja exibida na página. Porém é sempre bom destacar a importância da semantica e da acessibilidade do seu código em HTML, por isso sempre procure incluir pelo menos o atributo alt e inclua uma descrição sucinta porém objetiva do conteúdo da imagem.

Vale lembrar que a navegação no atributo src deve considerar a página em questão, usando a referência de dois pontos consecutivos (..) para voltar uma pasta nos diretórios do servidor e a barra (/) para entrar em um diretório.

Esse seria um exemplo de uso, considerando a estrutura deste projeto, para exibir em tela a foto do peril que está na pasta de assets disponível a partir da pasta raiz do projeto: <img src="../assets/foto-perfil.jpg" src="foto de uma mulher de cabelos pretos sorrindo"/>
