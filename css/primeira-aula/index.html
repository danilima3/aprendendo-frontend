Anotações relevantes

Descrição:

O CSS é responsável pela estilização da página, de modo a permitir inclur, cores, bordas, efeitos, transparências, transições, translações e muito mais aos elementos HTML com a intenção de se fazer um site moderno e bonito.

Formas de incluir estilos

Há três maneiras principais de incluir estilo nos elementos HTML, mas nessa aula vimos somente os dois primeiros:

  • estilo em linha (inline style): Se trata de incluir em uma tag HTML o atributo style e como valor as propriedades css display: flex; background: lightgreen;, por exemplo. Acontece que essa é uma forma que não é comumente usada em web sites, por diversos motivos relacionados à facilidade de manutenção do código, como por exemplo, caso precisasse mudar a cor de determinados elementos em um site completamente estilizado, precisaria procurar cada um desses elementos no HTML e alterá-los com a nova cor e isso precisaria se repetir com todos os outros atributos e elementos. Os maiores problemas causados aqui são repetição de código, o que ocasiona mais tempo para alterar uma simples estilização e aumenta a probabilidade de causar um erro na renderização, além do fato de perder determinadas funcionalidades do css (aprenderemos mais sobre elas em aulas futuras).
    Exemplo de uma estilização em linha
    <header style="display: flex; background: lightgreen;"> conteúdo do header </header>
  • estilo incorporado (embeded style): Nessa forma é possível estilizar a página através da inclusão da tag <style>, que é por convenção incluído dentro da tag <head>. Dentro dessa tag é possível selecionar o elemento passando o nome da tag ou algum atributo nela existente, como classes, id, data attributo ou qualquer outro. Isso resolve parte do problema de manutenção de código, que é causado no inline style, mas o não resolve forma completa.
    Exemplo de estilização incorporada
    <style> seletor-do-elemento { propriedade-css: valor da propriedade; } </style>
  • estilo importado (external style): Aprenderemos esse em uma aula futura.

Como selecionar elementos no css

Como mencionado acima, na parte do estilo incorporado, é possível selecionar o elemento HTML para estilizá-lo através do nome da tag ou através de algum atributo do mesmo. Essa só é considerada concluída quando abre e fecha chaves logo após o nome da tag ou do atributo e dentro das chaves é onde se inclui as propriedades css, para estilizar o elemento selecionado.
Selecionando através do nome da tag: Para selecionar desta forma, precisa apenas incluir o nome da tag seguido de chaves, conforme exemplo abaixo
<style> header { background: lightgreen; display: flex; } </style> Selecionando através do atributo class da tag: Para selecionar pelo atributo class da tag, precisa apenas incluir um ponto "." seguido pelo nome da class, como "cabecalho" por exemplo, e por fim abrindo e fechando chaves. Desta forma poderíamos substituir o nome da tag do exemplo anterior pela seleção da classe:
<style> .cabecalho { background: lightgreen; display: flex; } </style>

Aplicação das regras de estilo

Para a estilização escrita no css ser aplicada ela segue algumas regras primordias do css e, nessa aula, vimos algumas delas, como exemplificado na lista abaixo. Considere que a ordem definida na lista abaixo vai do mais forte para o mais fraco, ou seja, os primeiros itens tem prioridade para aplicação de suas regras.

  1. Expressão "!important"
    Se trata de incluir a expressão "!important" logo após o valor de uma propriedade, desta forma por exemplo: color: red !important. No entanto, cabe ressaltar, que o uso dessa expressão na declaração de qualquer propriedade é considerado má prática e não deve ser usada. Apenas foi mencionada na aula para tomar conhecimento e ter como opção caso não haja outra forma para que a regra de estilo desejada seja aplicada, porém é indicado sempre evitar o uso e se possível corrigir tudo o que está impedindo de definir a propriedade sem essa expressão.
  2. Style inline
    Qualquer estilo incluído diretamente no atributo style do elemento terá mais força do que qualquer seletor ou nível de força de seletor. Essa forma de selecionar o elemento também é considerada má prática se incluída diretamente no HTML, pelo fato de prejudicar a manutenção do código como já mencionado anteriormente.
  3. Nível de força do seletor
    O nível de força da seleção é definido pela quantidade de aninhamentos que um elemento pode ter, considerando tanto o nome da tag quanto o nome do atributo. Por exemplo, se é feita a seleção do mesmo elemento duas vezes, com a mesma quantidade de aninhamentos, eles terão a mesma força. Caso selecione o mesmo elemento onde em uma forma de seleção tenha pelo menos um aninhamento a mais que na outra, a seleção com mais aninhamentos terá mais força, mesmo que seja definida antes, exemplos:
    Elemento selecionado com mesmo nívevl de seleção
    <style> header { background: lightgreen; color: blue; } .cabecalho { background: lightgreen; color: red; } </style>
    Nesse caso, a regra aplicada será a segunda, pois os elementos possuem o mesmo nível de força de seleção, que é qualquer elemento header vs qualquer elemento com a classe .cabecalho, ou seja, nem a seleção pelo nome da tag nem a seleção pela classe estão aninhandadas (com outra tag ou classe precedendo ela).
    Elemento selecionado com níveis diferentes de força por aninhamento
    <style> header { background: lightgreen; color: red; } main header { background: lightgreen; color: blue; } </style>
    <style> main header { background: lightgreen; color: blue; } header { background: lightgreen; color: red; } </style> No exemplo acima, em ambos os casos a cor renderizada seria vermelha, pois a força de seleção o elemento que está aninhado main header é maior do que a força de seleção para o elemento solto header.Essa mesma regra se aplica caso em vez do nome da tag seja utilizada uma classe, por exemplo.
    Além do aninhamento, pode haver também a adição de um atributo, ou seja a seleção de um elemento pelo nome da classe ou por um atributo é mais fraca do que se a seleção juntar ambos, por exemplo, ao selecionar desta forma header.cabecalho se tem uma seleção mais forte do que se a seleção for desta forma header ou desta forma .cabecalho.
    Cabe ressaltar que não foi mencionado na aula, mas o atributo id, quando comparado com a seleção pelo nome da tag ou por qualquer outro atributo, é mais forte, ou seja ao fazer a seleção desta forma #titulo-1, é mais forte do que header ou .cabecalho.
  4. Ordem da definição das propriedades
    A sigla CSS significa Cascade Style Sheets, ou seja, folhas de estilo em cascata, o que significa que tudo que é incluído por último e que se refira à propriedade de um mesmo elemento selecionado, sobrescreverá o que foi definido antes, caso ambos os elemetos tenham o mesmo nível de força na seleção.
